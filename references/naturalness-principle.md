# 自然感法則

Google 和 Meta 的演算法有一個共同趨勢：越來越擅長偵測「刻意操控」的痕跡。AI 時代下，演算法越來越像人，越來越注重自然感，越來越討厭完美。

## 核心原理

### 演算法在抓什麼

兩個平台都在偵測同一件事：**不自然的完美**。

**Meta 端（Low Signal 偵測，US-20190095961-A1）**：
- AI 生成的 clickbait 開頭 → 被標記為 low signal
- 過度聳動的標題 → 被標記為 low signal
- 演算法在學習辨別「刻意吸引點擊但內容空洞」的模式

**Google 端（行為數據 + 真人評分）**：
- Navboost 看的是點擊後的行為——標題黨引來點擊但秒退 = 短點擊 = 負面信號
- Chrome 瀏覽數據直接餵排名系統（多位高層法庭親口證實）
- IS4 真人評分員手動打分，特別針對敏感詞、商業詞、YMYL 詞

### 越完美的 SEO 越不完美

傳統教的「完美」做法，現在反而是扣分項：

| 傳統做法 | 為什麼現在是問題 |
|---------|----------------|
| 關鍵詞堆滿 | 大品牌自然內容不會這樣寫，演算法看得出來 |
| 外鏈全部高 DR | 自然成長的網站一定有各種品質的連結混在一起 |
| Anchor text 全部精準匹配 | Ahrefs 研究 38 萬頁，排名前面的 exact match 接近零 |
| 零垃圾外鏈 | 等於告訴 Google「我的連結全是人工操控的」 |

### 自然感的三個實踐

**1. 關鍵詞：用同義語意詞自然鋪設**
- 刪掉重複堆疊 → 換成語意相近的自然表達
- 實測：傳產網站只做這兩件事，流量三個月起飛

**2. 外鏈：故意做得雜亂**
- 初期什麼連結都買一點——論壇、目錄、留言板、PBN、社群、側欄
- 高的低的全都要，佈局越亂越像自然成長

**3. Anchor text：精準匹配比例要接近零**
- 自然情境下別人連到你寫的是「這篇不錯」「參考這個」
- 八成以上 anchor 都是 exact match = 舉手告訴 Google 這些連結是自己做的

## Checklist：自然感檢查

### Threads 內容
□ 開頭是否自然？有沒有 AI 味的 clickbait？
□ 用詞是否像你平常說話？還是像 AI 生成的？
□ 有沒有過度使用公式化結構？（如「三個原因」「五步驟」）

### SEO 內容
□ 關鍵詞密度是否自然？有沒有堆疊嫌疑？
□ 外鏈結構是否看起來像自然成長？（不是全部高 DR）
□ Anchor text 是否多元？exact match 比例是否接近零？
□ 整體看起來像大品牌自然成長的樣子嗎？

## 核心觀點

> 「越完美的 SEO，其實越不完美。演算法越來越像人，人都不喜歡裝貨了，演算法更不會。」——darkseoking

> 「不管社交媒體還是搜尋，最終邏輯都是用不同參數找出使用者最喜歡的內容。AI 接入後，演算法越來越像人，自適應更新越來越快，漏洞越來越少。」

## Google 與 Meta 的共通邏輯

| 維度 | Google | Meta |
|------|--------|------|
| 品質判斷 | 長點擊 vs 短點擊（Navboost） | Expected Engagement Value |
| 操控偵測 | 不自然的外鏈/anchor pattern | Low signal / clickbait 偵測 |
| 權威系統 | E-E-A-T | Creator Embedding + Social Graph |
| 集群概念 | Topic cluster | 內容集群 |
| 時間衰減 | Navboost 記錄 13 個月 | Audience Affinity 有時間衰減 |
| 人為介入 | IS4 真人評分員 | （尚未有公開證據） |

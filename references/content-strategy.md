# 內容策略

發文前、中、後的關鍵決策。每個環節都有對應的演算法機制，做對了事半功倍，做錯了浪費好內容。

## 核心原理

### 爆文後的正確路徑（US9336553B2）

Meta 的推薦系統重視**內容多樣性**，防止同一帳號霸榜：
- 爆文後瘋狂產出同主題延伸文 → 直接降低 Ranking Score，甚至進不了 Candidate Pool
- 「趁熱度正高狂發延伸文」是錯的

**正確做法：面對同一群人、用不同角度切入**
```
爆文 → 感謝文（帶一點相關乾貨）→ 弱語意相關但同圈子的文
```

### 發文前測水溫（US8402094B2）

在發正式內容之前，先去別人文章底下留言測試：
- 如果留言被大量推送、成為原文主要曝光來源 → Audience Affinity 還在高點
- 此時發文幾乎不踩雷，不浪費精心準備的內容
- 如果留言反應冷 → 再等等，或先做互動暖場

### 外部連結評分（EP2977948A1 + US10268763B2）

「放外部連結就被限流」是錯的。實際有明確評分：
- **域名權威度**：該域名被其他使用者分享的次數越多 → 越不容易被限流
- **內容相關性**：連結跟貼文內容要有關聯
- 連結內容跟貼文無關（如留言賣課）→ 才會被限流

### 語意相似度保護（US10599774B1 + US10037333B2）

被抄襲時的演算法保護：
- Meta 有語意相似度比對系統，高相似度時壓後發那篇的曝光
- 先發有時間戳優勢
- 但不是即時，跨語言偵測也有漏洞
- 抄的人拿不走你的 Creator Embedding 跟 Audience Affinity

### Second-Chance 機制（US10635732B2）

沒爆的文有可能被系統自動重新推送：
- 這是**系統觸發**的，不是你刪了重發能觸發
- 刪文重發 → 相似度偵測判重複 + 所有互動信號歸零 = 更慘

## Checklist：發文決策

### 發文前
□ 去同領域留言測過水溫了嗎？Affinity 還在高點嗎？
□ 內容跟帳號定位一致嗎？（不偏離 Creator Embedding）
□ 距離上篇至少 24 小時了嗎？
□ 開頭自然嗎？有沒有 clickbait / AI 味？
□ 如果有外部連結——域名被分享過嗎？跟內容相關嗎？

### 發文後
□ 前 3 小時有在積極回覆互動嗎？
□ 互動是否已經榨乾？（留言都回了？每則都按讚了？）
□ 準備休息 24-48 小時再發下一篇
□ 下一篇有沒有換角度？（不是同主題延伸文）

### 貼文沒爆時
□ 不要刪文重發（唯一例外：10 分鐘內、零互動、有嚴重錯誤）
□ 善用編輯功能，小改不影響曝光
□ 分析為什麼沒爆：開頭問題？主題偏離？撞到前一篇 freshness？
□ 用不同角度重新寫一篇，比刪文重發有效太多

## 常見迷思破解

### 「刪文重發可以重來？」
99% 情況沒用。內容相似度偵測判定重複、所有累積信號歸零。正確做法：找到沒爆的原因，用不同角度重寫。

### 「放外部連結就被限流？」
大錯特錯。域名被多人分享過 + 跟內容相關 = 不會被限流。被限流的是連結跟貼文無關的情況。

### 「被抄襲了怎麼辦？」
持續產出比糾結一篇被抄更划算。抄的人拿不走你的語意標籤和受眾信任。越多人抄反而變相幫你宣傳。

## 實戰案例

darkseoking 的爆文後路徑：
```
專利爆文 → 感謝文（帶一點專利乾貨）→ SEO 演算法文（弱語意相關但同圈子）
```
面對同一群人、用不同角度切入，就不會被多樣性機制懲罰。
